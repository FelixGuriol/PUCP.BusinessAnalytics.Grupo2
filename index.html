Proyecto: Analytics and Data Visualization with Power BI for ABC Glass

📌 Descripción General del Proyecto

Este proyecto busca implementar un KPI, un modelo de datos, un dashboard y reportes de resultados de producción para una empresa ficticia del sector automotriz.

1. Objetivos y contexto

🎯 Objetivos Principales

  • Definir un KPI del negocio basado en el libro Key Performance Indicators (KPI): The 75 Measures Every Manager Needs to Know de Bernard Marr, incluyendo su concepto, importancia, fórmula, umbrales e impacto económico.
  • Describir un modelo de datos que permita calcular el KPI definido.
  • Elaborar un caso de negocio que ilustre cómo una estrategia puede corregir una desviación del KPI según sus umbrales.
  • Desarrollar un dashboard del KPI.
  • Elaborar reportes del KPI y otros indicadores asociados para responder a 10 preguntas relevantes de análisis del negocio.

🏭 Información del negocio

ABC Glass

ABC Glass es una empresa ficticia dedicada al diseño y fabricación de vidrio automotriz para fabricantes de vehículos.

La compañía desarrolla soluciones de vidrio orientadas a la seguridad, funcionalidad y facilidad de ensamblaje en vehículos modernos.

La empresa ficticia opera como proveedor Tier 1 de componentes automotrices, suministrando vidrio a fabricantes OEM de vehículos de diversos cuatro países.

4. Herramientas Tecnológicas y Arquitectura

🔧 Herramientas Tecnológicas

Para el desarrollo de este proyecto se emplearon diversas herramientas de Microsoft Cloud y aplicaciones de apoyo que permitieron integrar, transformar y visualizar los datos de manera eficiente:

Tecnología Función Principal
Azure SQL Almacenamiento relacional en la nube para centralizar y normalizar los datos.
Microsoft Fabric Procesamiento, integración y orquestación de datos mediante pipelines y notebooks.
Power BI Creación de dashboards interactivos y reportes dinámicos para el análisis de indicadores.
Power Apps Registro operativo de pedidos, entregas, inventario y actualización de tablas maestras.
Lucidchart Diseño de diagramas de flujo y modelos de arquitectura para documentar los procesos.
Power Automate Automatización de reportes y envío de información periódica vía correo electrónico.

🏗️ Arquitectura del Proyecto

La arquitectura del proyecto fue diseñada para integrar múltiples herramientas en la nube con el fin de automatizar y facilitar la medición del KPI. Esta solución combina el uso de una base de datos en SQL Azure, procesamiento automático en Microsoft Fabric, aplicaciones operativas en Power Apps, flujos de automatización con Power Automate y visualizaciones con Power BI.

📁 Data Sources

El origen de los datos son archivos SQL que contienen información transaccional del negocio. Estos archivos constituyen el punto de entrada de la arquitectura y alimentan el proceso de ingesta hacia Azure SQL Database.

🗄️ Azure SQL Database

Los datos son procesados mediante ETL e ingresados en Azure SQL Database, donde se normalizan y estructuran en un modelo relacional en la nube, sirviendo como fuente central para los procesos de transformación posteriores.

📱 Power Apps

Power Apps permite a los usuarios operativos registrar y actualizar información directamente desde formularios conectados a la arquitectura, alimentando el Fabric Data Warehouse con datos capturados en tiempo real.

⚙️ Data Flow & Pipeline (ETL)

Los componentes Data Flow y Pipeline de Microsoft Fabric orquestan la transformación y carga de datos desde Azure SQL Database hacia el Fabric Data Warehouse, automatizando la ingesta y garantizando la calidad de la información.

🏢 Fabric Data Warehouse

El Fabric Data Warehouse es el repositorio central que consolida datos provenientes de los pipelines, Power Apps, Python y Web Services. Desde aquí se sirve la información procesada hacia Power BI para su análisis.

🐍 Python & Web Service

Scripts de Python y Web Services complementan la arquitectura con procesamiento personalizado e integraciones externas, enviando datos adicionales al Fabric Data Warehouse para enriquecer el análisis.

📊 Power BI

Power BI se conecta al Fabric Data Warehouse para transformar los datos consolidados en dashboards interactivos y reportes dinámicos, facilitando el análisis de indicadores clave para los usuarios finales.

👥 End User

Los usuarios finales acceden a los reportes e insights generados en Power BI para la toma de decisiones basada en datos, siendo el punto de consumo final de toda la arquitectura de datos.

🗄️ Data Layer – Capa de Datos

En esta capa se diseñaron y consolidaron las bases de datos necesarias para el análisis del KPI. Estas tablas permiten capturar, organizar y relacionar la información proveniente de pedidos, producciones, defectos y datos maestros, asegurando consistencia y trazabilidad en todo el flujo de información.

📊 Modelo de datos en Power BI

El modelo de datos fue construido en Power BI a partir de las tablas publicadas desde el entorno de Microsoft Fabric. Las relaciones se establecen entre las tablas de hechos y dimensiones mediante claves primarias y foráneas. La estructura responde a un esquema en estrella, permitiendo eficiencia en los análisis.

Dentro de este modelo, se incluye una tabla de fechas (Calendario) para análisis temporales y una tabla de medidas que contiene indicadores calculados como el porcentaje del KPI, cantidad producida, cantidad pedida, entre otros.

Este modelo es administrado por el Analista de Inteligencia de Negocios, quien se encarga de estructurar y validar los datos antes de la construcción del dashboard.

⚙️ ETL Layer – Capa de Extracción, Transformación y Carga

En esta capa se gestionan los procesos de ingesta, transformación y carga de datos, asegurando que la información esté siempre actualizada y disponible para los diferentes análisis del proyecto. Aquí se integran el notebook en PySpark y el pipeline en Microsoft Fabric, que trabajan de manera conjunta para garantizar eficiencia, trazabilidad y control en el flujo de datos.

📁 Origen de Datos

Los datos utilizados en esta etapa provienen de tablas consolidadas en Azure SQL y replicadas en Microsoft Fabric. Estas incluyen información de pedidos, producciones, defectos y datos maestros, las cuales constituyen la base sobre la cual se ejecutan los procesos de notebook y pipeline.

Notebooks en PySpark

Una parte clave del proceso automatizado fue el desarrollo de 2 notebooks en PySpark, que se ejecutan al final del pipeline diario en Microsoft Fabric. Estos notebooks tienen como propósito principal consultar la producción hecha en el día y enviar dicha información automáticamente por correo electrónico y WhatsApp al usuario.

Flujo del Proceso

  • Consulta de Datos: Los notebooks se conectan al Data Warehouse en Fabric y realizan una consulta SQL para contar los pedidos registrados en la fecha actual.
  • Formateo de Resultados: La información obtenida se transforma en un formato legible para que pueda ser interpretada rápidamente por los usuarios finales.
  • Notificación Vía Correo: Se envía un correo automático con los resultados, utilizando un servidor SMTP configurado en Fabric.
  • Notificación Vía WhatsApp: Se envía un mensaje de WhatsApp automático con los resultados, utilizando el API de Twilio configurado en Fabric.

Pipeline en Microsoft Fabric

La automatización del flujo de datos se gestiona mediante un pipeline en Microsoft Fabric llamado pl_tablas, configurado para actualizar diariamente las tablas del Data Warehouse y ejecutar los notebooks PySpark al finalizar. El pipeline sigue una estructura secuencial donde cada tabla es procesada individualmente mediante Data Flows.

Característica Detalle
Nombre pl_tablas
Frecuencia de Ejecución Diario a las 8:00 PM
Herramienta Usada Microsoft Fabric - Data Factory
Tipo de Tareas
  • Actualiza las tablas importadas desde Azure SQL hacia Fabric Data Warehouse.
  • Ejecuta automáticamente los Notebooks de PySpark (correo y WhatsApp) al finalizar.
  • No realiza carga desde fuentes externas, sino sincronización y limpieza de tablas ya integradas.

5. Conclusiones

El proyecto "Analytics and Data Visualization with Power BI for ABC Glass" logró integrar exitosamente diversas herramientas tecnológicas para transformar por completo la gestión operativa y analítica del proceso de producción y seguimiento de pedidos. Desde el registro y validación de pedidos y producciones en Power Apps hasta la actualización automatizada del modelo en Microsoft Fabric, se construyó un flujo sólido y eficiente que garantiza información actualizada en tiempo real.

A través de un modelo de datos estructurado bajo un esquema en estrella y la automatización de cálculos en notebooks PySpark, se permitió calcular de forma precisa el KPI definido para el negocio, asegurando trazabilidad y confiabilidad en cada etapa del proceso. El dashboard en Power BI facilitó el análisis visual, permitiendo identificar oportunidades de mejora por modelo de vehículo, tipo de vidrio, cliente y país, así como comprender la relación entre la producción y los pedidos a lo largo del tiempo.

Además, se estableció un sistema automatizado de notificación diaria mediante notebooks PySpark que distribuyen alertas de nuevos pedidos por correo electrónico y WhatsApp a los responsables sin intervención manual, lo que demuestra una clara apuesta por la eficiencia y la continuidad operativa. Todo este ecosistema digital ha permitido simular cómo una empresa manufacturera del sector automotriz puede adoptar decisiones basadas en datos confiables, mejorando tanto el control de producción como el servicio a sus clientes OEM. En resumen, este proyecto no solo ha cumplido con el objetivo de medir y visualizar el KPI de producción, sino que también ha demostrado cómo la integración tecnológica puede aportar valor real, escalable y sostenible en entornos industriales de alta exigencia como el de la manufactura de vidrio automotriz.
